# 1.2. Módulo Artefato Generalista

## Introdução
Os artefatos generalistas são instrumentos de documentação e planejamento que não pertencem exclusivamente a uma única metodologia, mas que fornecem a base estratégica para o desenvolvimento do software. No contexto do **MinhaTirinha**, esses artefatos ajudam a alinhar as expectativas da equipe e a definir o escopo do projeto de forma clara e acessível.

## Metodologia
A equipe utilizou o framework **5W2H** para o planejamento tático e a criação de um **Glossário** para padronização da linguagem técnica e de domínio. Além disso, para a organização das tarefas e fluxo de trabalho, utilizamos o **Trello**, aplicando conceitos de gestão visual para garantir a transparência do progresso entre os 10 integrantes. Todas as reuniões do grupo foram gravadas e armazenadas no ambiente compartilhado do Teams para consulta posterior e validação dos requisitos

## Principais Artefatos Utilizados

### 5W2H
O 5W2H é uma ferramenta de gestão que utiliza sete perguntas para definir as ações necessárias no projeto.

| Pergunta | Resposta |
| :--- | :--- |
| **What (O quê?)** | Desenvolvimento do aplicativo **MinhaTirinha** para pintura de HQs. |
| **Why (Por quê?)** | Auxiliar no combate ao estresse e ansiedade através da arteterapia digital. |
| **Where (Onde?)** | Repositório no GitHub e testes mobile via Expo Go. |
| **When (Quando?)** | Semestre letivo de 2026.1 (março a julho de 2026). |
| **Who (Quem?)** | 10 estudantes da disciplina de Arquitetura e Desenho de Software (Grupo 08). |
| **How (Como?)** | Utilizando React Native, Figma para prototipação e metodologias ágeis (Scrum/XP). |
| **How Much (Quanto?)** | Custo financeiro zero, utilizando ferramentas gratuitas e open-source. |

# Glossário

O glossário tem como objetivo nivelar o conhecimento da equipe sobre os termos técnicos, de design e de negócio utilizados no projeto **MinhaTirinha**, garantindo a rastreabilidade e a clareza na comunicação.

<div align="center">

| Termo | Definição | Categoria |
| :--- | :--- | :--- |
| **Acessibilidade** | Práticas de design e código que garantem que o app seja usado por pessoas com diferentes capacidades. | Design |
| **Arteterapia** | Uso de recursos artísticos e criativos com fins terapêuticos e para expressão de sentimentos. | Domínio |
| **Backlog** | Lista ordenada de todas as funcionalidades, requisitos e melhorias necessárias para o produto. | Processo |
| **Balão de Fala** | Elemento gráfico da HQ que contém diálogos, desbloqueado no app somente após a pintura total do quadrinho. | Funcionalidade |
| **BPMN** | *Business Process Model and Notation*. Notação padrão para modelagem de processos de negócio. | Processo |
| **Code Review** | Revisão sistemática de código feita por um colega para garantir qualidade e padrões do grupo. | Engenharia |
| **Deploy** | Processo de colocar a aplicação ou documentação em um ambiente de execução (ex: GitHub Pages). | Tecnologia |
| **Domínio** | A área de conhecimento específica sobre a qual o software está sendo desenvolvido (HQs e Bem-estar). | Negócio |
| **Expo Go** | Aplicativo que permite rodar e testar o código React Native em dispositivos móveis em tempo real. | Tecnologia |
| **Gateway** | Ponto de decisão em um fluxo BPMN que determina o caminho a ser seguido no processo. | Processo |
| **HQ (Gibi)** | História em quadrinhos. O formato narrativo base utilizado na aplicação. | Domínio |
| **Pair Programming** | Técnica de engenharia (XP) onde dois desenvolvedores trabalham juntos no mesmo código. | Engenharia |
| **Pull Request (PR)** | Solicitação para integrar mudanças de código de uma branch para a branch principal no GitHub. | Tecnologia |
| **Rich Picture** | Técnica visual (Brainstorming) para representar a visão geral de um problema e a jornada do usuário. | Design |
| **Storyboard** | Sequência de ilustrações que descreve a jornada e a experiência de uso do personagem no sistema. | Design |
| **Tons Pastéis** | Paleta de cores de baixa saturação utilizada para promover calma, foco e alívio de estresse. | Design |

</div>
