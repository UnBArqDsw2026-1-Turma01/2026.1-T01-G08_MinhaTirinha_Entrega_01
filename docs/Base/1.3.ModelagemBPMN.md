# 1.3. Módulo Modelagem BPMN

## Introdução
A modelagem BPMN (*Business Process Model and Notation*) é uma notação padrão para representar os processos de trabalho de forma visual e compreensível, facilitando o alinhamento entre a equipe e os stakeholders. Para o desenvolvimento do **MinhaTirinha**, o Grupo 08 adotou uma abordagem baseada em metodologias ágeis.

## Metodologia
O grupo optou por uma metodologia híbrida, integrando o framework **Scrum** para a gestão e planejamento das atividades com práticas do **Extreme Programming (XP)** para garantir a qualidade da engenharia de software:

* **Scrum:** Utilizamos ciclos de desenvolvimento de duas semanas (**Sprints**), iniciados com o **Sprint Planning** para definir o que será feito e encerrados com a **Sprint Review** e **Retrospective** para avaliar o incremento e o processo.
* **XP:** Aplicamos práticas de **Programação em Par** (*Pair Programming*) para a codificação e revisão constante de código através de **Pull Requests** no GitHub.
* **Gestão Visual:** Utilizamos o **Trello** para o controle do fluxo de tarefas (Backlog, Doing, Done).
